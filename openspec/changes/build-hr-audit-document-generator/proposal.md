## Why

汽车零配件生产企业在面临客户审核（通常高于 IATF16949 标准）和体系审核时，需要在短时间内整理出大量格式统一、仅基本信息不同的人力资源审核资料（入职须知、登记表、合同、培训记录等）。承担这项工作的审核老师目前主要靠手工打印、填写、整理，重复劳动量极大且容易出错。本变更要为审核老师提供一个 Web 系统：上传真实员工花名册与可选签名后，按内置模版和体系规则自动生成整套人力资源审核资料包。

## What Changes

- 新增 Web 上传界面（分模块），人力模块支持上传员工花名册与可选的员工手写签名照片；签名照片经过抠底处理为电子签名。
- 新增资料生成引擎：依据花名册与体系规则，自动生成每名员工的入职/转正/培训档案，以及公司级培训计划与培训签到记录。
- 新增资料包下载界面：列出生成的文件清单，支持打包 ZIP 下载。
- 新增模版管理界面：内置全部表单模版，可在线编辑保存、维护 `{{}}` 占位符，并预览真实数据替换后的效果。
- 新增「个人培训记录表（一人一年一份）」模版，从年度培训计划与各场培训记录中按员工聚合生成。
- 修正与扩展现有模版的占位符与花名册字段，使所有 `{{}}` 占位的数据来源在员工主数据中可被解析。
- 签名为非必填：上传则全局使用电子签名图片，未上传则使用非打印体文本签名，不阻塞生成。
- 日期排布由系统按单休 + 日历真实排算（不出现 2 月 31 日等非法日期），用户无需关心内部算法。

## Capabilities

### New Capabilities
- `client-data-intake`: 客户基础资料与员工花名册上传、字段校验与标准化。
- `signature-processing`: 可选签名照片上传、抠底生成电子签名，以及未上传时的非打印体签名回退。
- `document-template-management`: 模版的在线浏览、编辑保存、占位符维护与真实数据预览。
- `hr-document-generation`: 依据员工档案规则（入职/转正/技能/满意度）生成人力资源资料。
- `training-record-generation`: 依据年度培训计划生成培训计划表、培训签到记录与个人培训记录表，并按培训面向对象匹配员工。
- `audit-package-export`: 生成文件清单、组织目录结构并打包为 ZIP 资料包下载。

## Impact

- 新增前端 Web 界面（上传页、下载页、模版管理页）与后端服务（生成引擎、模版渲染、ZIP 打包）。
- 依赖现有 `template/人力资源/**` 模版资产与 `json文件/training-plan-comprehensive.json` 培训计划数据。
- 新增员工主数据模型、培训面向对象到员工的匹配映射、工作日/单休日历计算能力。
- 需要修正 `template/人力资源/岗位安全检查表/template.html` 中非标准占位符 `{position_safety.month_label}`，并扩展花名册字段集合。
- 涉及真实员工个人信息与签名，需考虑授权、存储与可追溯性。
